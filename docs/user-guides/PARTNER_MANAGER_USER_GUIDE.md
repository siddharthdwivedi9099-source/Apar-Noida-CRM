# Partner Manager User Guide

## Purpose

This guide explains how partner managers use the Partner channel workspace introduced in Phase 13.

## Accessing the workspace

Open **Partners** from the navigation. The module appears for roles with Partners access (for example the seeded Partner Manager and Reseller Manager roles) and when the tenant has the Partners module enabled.

## Partner dashboard

The top of the workspace shows channel metrics for your visibility scope:
- Total partners and active partners
- Onboarding in progress
- Registered deal count and registered deal value
- Won deals, tier coverage, and your current scope (mine / team / all)

## Partner list

The partner list shows each partner's tier, type, status, region/territory, owner, onboarding progress, and deal count. Select a partner to open its detail panel.

## Creating a partner

1. Click **Add partner** (requires create permission).
2. Enter the partner name and choose type, tier, status, and onboarding status.
3. Optionally link a CRM account, assign a partner owner, and set region, territory, and an agreement reference.
4. Optionally add onboarding checklist tasks, one per line.
5. Submit to create the partner; the new record opens automatically.

## Partner detail, onboarding, and deals

The detail panel shows:
- Profile, owner, onboarding status, linked account, and agreement reference
- Partner contacts (primary contact highlighted)
- **Onboarding checklist** — use **Advance** to move a task through pending → in progress → completed
- **Registered deals** — review stage, customer, amount, and expected close date
- Register a new deal with the inline form (name, stage, customer, amount, optional linked opportunity)
- Permission-aware AI placeholders (fit score, performance summary, action plan, churn risk, conflict detection)

## Permission behavior

- Users without Partners access do not see the module in navigation.
- Users without create permission cannot open the create form.
- Users without edit permission cannot advance onboarding tasks or register deals.
- Users without assign permission cannot reassign partner ownership.

## Current limits

- Enablement assets, training linkage, and support ticket linkage are placeholders.
- AI placeholder actions are visible but deferred until the AI Gateway phase.
- There is no external partner portal in this phase.
